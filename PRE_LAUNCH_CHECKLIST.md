# ✅ قائمة الفحص النهائية — Heru Store v3 (Pre-Launch Checklist)

### 🔐 الأمان (Security)
- [ ] `SUPABASE_SERVICE_ROLE_KEY` غير موجود إطلاقاً في الأكواد الجبهية (فقط مخزن في `.env.local`)
- [ ] التحقق من أن `.env.local` موجود داخل ملف `.gitignore` لمنع تسريبه
- [ ] تم تفعيل `RLS` (Row Level Security) على كل الجداول في قاعدة بيانات `Supabase`
- [ ] لوحة تحكم الإدارة `/admin` محمية بالكامل عبر `middleware`
- [ ] لا توجد أوامر `console.log` تُظهر أو تُسّرب بيانات حساسة في بيئة الـ `production`

### 🗄️ قاعدة البيانات (Database)
- [ ] أوامر الـ `schema.sql` الخاصة ببناء الجداول قد نُفّذت بنجاح في `Supabase`
- [ ] متسلسلة أرقام الطلب `SEQUENCE order_number_seq` تعمل بنجاح (اختبر ذلك بإنشاء طلب وهمي)
- [ ] تم إنشاء حساب مدير النظام (Admin user) في قسم `Supabase Auth`
- [ ] حاويات رفع ملفات الصور (`Storage buckets`): `product-images` و`category-images` موجودة وصلاحياتها `public`
- [ ] تم إدخال إعدادات المتجر الأساسية (Store Settings) في جدول `settings`

### 📱 المحمول (Mobile Responsiveness)
- [ ] الموقع يعمل بتوافق بصري سليم على شاشات `iPhone SE` بعرض (375px)
- [ ] الموقع يعمل بتوافق بصري سليم على شاشات `Samsung Galaxy` الأصغر بعرض (360px)
- [ ] لا يوجد أي خلل بصري من نوع الـ `overflow` الأفقي المسبب لانزياح الشاشة
- [ ] قائمة المحمول الجانبية (`Mobile menu`) تعمل بسلاسة بدون ظهور عشوائي أو تداخلات
- [ ] جميع الأزرار قابلة للنقر بسهولة ولا تقل مساحتها عن `44px minimum touch target`

### ⚡ الأداء (Performance)
- [ ] تم اختبار الأداء عبر `Lighthouse Performance` محققاً تقييم أعلى من **85**
- [ ] تحميل العنصر الأكبر بالصفحة (`LCP`) لا يتعدى حاجز التخطي المقدر بـ **2.5s**
- [ ] لا يوجد أي تزحزح أو تخبّط في عناصر الشاشة أثناء التحميل (`CLS = 0`)
- [ ] جميع صور العناصر موُضحة بقياسات ثابتة (`width` + `height`) لمنع القفزات التصميمية
- [ ] صورة الغلاف الرئيسية (`Hero image`) تحتوي على خاصية `priority={true}` للتحميل المبكر

### 🌐 تهيئة محركات البحث (SEO)
- [ ] كل صفحة من صفحات المنتجات تولّد معرّفاتها الوصفية عبر `generateMetadata`
- [ ] خريطة الموقع (`sitemap.xml`) تعمل بنشاط على مسار (`/sitemap.xml`)
- [ ] موجهات العناكب (`robots.txt`) تعمل على مسار (`/robots.txt`)
- [ ] الأوسمة التسويقية (`OG tags`) صحيحة ودقيقة (قُم باختبارها عبر أداة `Facebook Debugger`)

### 🛒 رحلة المستخدم (User Journey Checkout Test)
- [ ] الصفحة الرئيسية تُحمّل بصورة سريعة وتُظهر حزم المنتجات بسلاسة 
- [ ] أداة البحث في المتجر تعمل وتستجيب بفاعلية
- [ ] صفحة التفاصيل الخاصة بالمنتج تُظهر الصور والسعر بشكل سليم تماماً
- [ ] زر "اشتر الآن" السريع، يفتح تطبيق `WhatsApp` محملاً بنص الرسالة الصحيح
- [ ] إضافة منتج لعربة التسوق (`Cart`) تعمل، وعداد العربة يُحدّث
- [ ] التحقق من قيود نموذج شحن الطلب (`Checkout form validation`) يعمل ويرفض الإدخالات الخاطئة
- [ ] إجراءات الخادم الخلفية (`Server Action`) تُنشئ وتهيكل الطلب وتقوم برفعه بامتياز في `Supabase`
- [ ] تطبيق `WhatsApp` يُفتح آلياً بعد نجاح خطوة إنشاء الطلب (`checkout`)
- [ ] صفحة تأكيد استلام الطلب تُعلم المستخدم برسالة نجاح وتُظهر رقم طلبه بوضوح

### 👩‍💼 لوحة تحكم الإدارة (Admin Panel)
- [ ] واجهة تسجيل الدخول تعمل وتستجيب للصلاحيات
- [ ] عملية إضافة منتج جديد + رفع صورة للمنتج تعمل بسلاسة وبدون أخطاء
- [ ] الصورة المرفوعة تظهر فوراً وبالدقة المأمولة بمتجر العملاء
- [ ] تغيير حالات سير الطلب (مُعلق، منجز، إلخ) يعمل وتحدث القاعدة
- [ ] الإعدادات القابلة للتغيير تُحفظ وتُطبّق على واجهات المتجر على الفور

### ⚖️ الشؤون القانونية والسياسات (Legal)
- [ ] صفحة `سياسة الخصوصية` متوفرة وتحوي محتوى قانونيّ حقيقي يوضّح آليه التعامل
- [ ] صفحة `شروط الاستخدام` متوفرة وتحوي محتوى الشروط والخدمات
- [ ] `سياسة الإرجاع` والاستبدال موضحة للمشتري درءاً للإلتباس
- [ ] رقم `WhatsApp` المربوط بالمبيعات صحيح وثابت في كل أقطاب ظهور المتجر

### 🚀 خطوات الإطلاق الخاتمة (Launch)
- [ ] الدومين الفعلي والدائم (Domain) تم ربطه بمنصة `Vercel`
- [ ] شهادة التشفير الآمن `HTTPS` مفعلة وتعمل (عادةً ما تقوم `Vercel` بها تلقائياً)
- [ ] أداة تحليلات `Google Analytics` مقعّلة بنجاح وتستقطب بيانات الزيارات الحية
- [ ] أداة `Sentry` مفعلّة وتتلقى بصمت أخطاء النظام الجاهز للمتابعة والتطوير المستمر
